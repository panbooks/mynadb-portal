MynaDB
Time Series Database

Your infrastructure is talking. Start listening.

Listen. Store. Monitor.

A blazing-fast time series database built with Rust. Ingest via InfluxDB Line Protocol or OpenTelemetry, store in columnar Parquet with Apache Iceberg support, and query with SQL powered by Apache DataFusion. Multi-protocol ingestion, streaming queries, and Grafana-native dashboards.

▶ Get Started 📖 Read the Docs 
# Write data using InfluxDB Line Protocol
curl -X POST http://localhost:7000/write/mydb \
  -d 'cpu,host=server01,region=us-west usage=75.5,temp=45'

# Query with SQL
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) FROM cpu GROUP BY host'

Quick Start

Start listening in 60 seconds.

bash 
# 1. Build and run
cargo build --release
./target/release/mynadb

# 2. Write some data
curl -X POST http://localhost:7000/write/mydb -d '
cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-east usage=82.3,temp=50
memory,host=server01 used=16.5,total=32.0
'

# 3. Query immediately! (No flush needed - Memtable provides instant visibility)
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) as avg_cpu FROM cpu GROUP BY host'

# Result:
# [{"host":"server01","avg_cpu":75.5},{"host":"server02","avg_cpu":82.3}]

# 4. Time series aggregation with GROUP BY time()
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) FROM cpu GROUP BY time(5m), host'
