What is MynaDB?

Like the myna bird known for its remarkable ability to listen and perfectly recall sounds, MynaDB captures every signal from your infrastructure with perfect fidelity. It's a time series database designed for high-throughput ingestion and fast analytical queries, combining the best of modern data infrastructure:

  • InfluxDB Line Protocol & OpenTelemetry - Multi-protocol ingestion for metrics, logs, and traces
  • Apache Parquet & Iceberg - Columnar storage with multi-engine catalog support
  • Apache DataFusion - Full SQL with forecasting UDFs and geospatial functions
  • Apache Arrow Flight - High-performance gRPC streaming for data transfer
  • Grafana & Prometheus - Native monitoring integration and metrics export
  • Rust - Memory-safe, zero-cost abstractions for maximum performance

Smart Caching

Multi-level caching: metadata, query results, and Parquet files on SSD.

Quick Example

bash 
# Write metrics using InfluxDB Line Protocol
curl -X POST http://localhost:7000/write/mydb -d '
cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-east usage=82.3,temp=50
memory,host=server01 used=16.5,total=32.0
'

# Query immediately - no flush needed! (Memtable provides instant visibility)
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) as avg_cpu FROM cpu GROUP BY host'

# Time series aggregation with GROUP BY time()
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT AVG(usage) FROM cpu GROUP BY time(5m), host'
