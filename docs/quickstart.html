Quick Start Guide

Start listening to your infrastructure in 60 seconds. This guide covers building, running, writing data, and querying with MynaDB.

Prerequisites

  • Rust 1.70+ (install via rustup)
  • curl (for testing the API)

Step 1: Build and Run

bash 
# Clone the repository
git clone https://github.com/mynadb/mynadb.git
cd mynadb

# Build in release mode
cargo build --release

# Run the server
./target/release/mynadb

You should see output like:

MynaDB starting...
HTTP server listening on 0.0.0.0:7000
Arrow Flight listening on 0.0.0.0:7001
Ready to accept connections

Step 2: Write Data

MynaDB accepts data in InfluxDB Line Protocol format:

Line Protocol Format 
<measurement>,<tag_key>=<tag_value> <field_key>=<field_value> [timestamp]

Write some sample data:

bash 
# Write CPU and memory metrics
curl -X POST http://localhost:7000/write/mydb -d '
cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-west usage=45.2,temp=38
cpu,host=server03,region=us-east usage=89.1,temp=52
memory,host=server01 used=16.5,total=32.0
memory,host=server02 used=24.8,total=64.0
memory,host=server03 used=12.2,total=32.0
'
💡 About the Database

The database (mydb) is automatically created when you first write to it. No need to create it explicitly!

Step 3: Query with SQL

⚡ Instant Query Visibility

No need to flush! MynaDB's Memtable provides immediate query visibility for fresh writes. The query engine automatically combines data from memory (unflushed) and Parquet files (flushed).

Now query your data using standard SQL:

Basic Select

bash 
# Select all CPU data
curl 'http://localhost:7000/query/mydb' -d 'SELECT * FROM cpu'

Aggregation

bash 
# Average CPU usage by host
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) as avg_cpu FROM cpu GROUP BY host'

Result:

[
  {"host": "server01", "avg_cpu": 75.5},
  {"host": "server02", "avg_cpu": 45.2},
  {"host": "server03", "avg_cpu": 89.1}
]

Filtering

bash 
# High CPU usage servers
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, usage FROM cpu WHERE usage > 70 ORDER BY usage DESC'

JOIN Queries

bash 
# Join CPU and memory data
curl 'http://localhost:7000/query/mydb' -d "
SELECT c.host, c.usage as cpu_usage, m.used as memory_used
FROM cpu c
JOIN memory m ON c.host = m.host
"

Step 4: List Tables

See what tables are available in your database:

bash 
curl http://localhost:7000/tables/mydb

Result:

{
  "catalog": "mydb",
  "tables": ["mydb.default.cpu", "mydb.default.memory"],
  "count": 2
}

Summary

You've successfully:

  • ✓ Built and started MynaDB
  • ✓ Written time series data using Line Protocol
  • ✓ Queried data immediately (no flush needed!)
  • ✓ Used SQL with aggregations and JOINs
  • ✓ Listed available tables
