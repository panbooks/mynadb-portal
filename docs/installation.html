Installation

This guide covers multiple ways to install MynaDB: pre-built binaries, building from source, Docker containers, and setting up as a system service.

System Requirements

Hardware

Component Minimum Recommended
CPU 2 cores 4+ cores
Memory 2 GB 8+ GB
Storage 10 GB SSD 100+ GB NVMe SSD
Network 100 Mbps 1 Gbps+

Software

  • Operating System: Linux (Ubuntu 20.04+, Debian 11+, RHEL 8+, etc.)
  • Architecture: x86_64 (AMD64) or ARM64
  • Rust: 1.75+ (for building from source)

Binary Release

The easiest way to install MynaDB is using pre-built binaries.

Download Latest Release

bash 
# Download the latest release
curl -LO https://github.com/mynadb/mynadb/releases/latest/download/mynadb-linux-amd64.tar.gz

# Extract the archive
tar -xzf mynadb-linux-amd64.tar.gz

# Move binary to /usr/local/bin
sudo mv mynadb /usr/local/bin/

# Verify installation
mynadb --version

ARM64 (Apple Silicon, AWS Graviton)

bash 
# Download ARM64 version
curl -LO https://github.com/mynadb/mynadb/releases/latest/download/mynadb-linux-arm64.tar.gz

# Extract and install
tar -xzf mynadb-linux-arm64.tar.gz
sudo mv mynadb /usr/local/bin/

Build from Source

Building from source gives you the latest features and allows customization.

Prerequisites

bash 
# Install Rust (if not already installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Reload shell configuration
source $HOME/.cargo/env

# Verify Rust installation
rustc --version  # Should be 1.75+

Clone and Build

bash 
# Clone the repository
git clone https://github.com/mynadb/mynadb.git
cd mynadb

# Build release binary
cargo build --release

# Binary is now at target/release/mynadb
./target/release/mynadb --version

# Optional: Install to system path
sudo cp target/release/mynadb /usr/local/bin/

Build Script

Use the provided build script to create a distributable archive:

bash 
# Build and create release archive
./build.sh

# Output: mynadb-vX.X.X-linux-amd64.tar.gz

Docker

Run MynaDB in a Docker container for easy deployment and isolation.

Pull Official Image

bash 
# Pull the latest image
docker pull ghcr.io/mynadb/mynadb:latest

# Run with default settings
docker run -d \
  --name mynadb \
  -p 7000:7000 \
  -p 7001:7001 \
  -v stats-data:/data \
  ghcr.io/mynadb/mynadb:latest

Docker Compose

For production deployments, use Docker Compose:

yaml 
# docker-compose.yml
version: '3.8'

services:
  mynadb:
    image: ghcr.io/mynadb/mynadb:latest
    container_name: mynadb
    restart: unless-stopped
    ports:
      - "7000:7000"   # HTTP API
      - "7001:7001"   # Arrow Flight
    volumes:
      - ./data:/data
      - ./config.toml:/etc/mynadb/config.toml:ro
    environment:
      - RUST_LOG=info
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:7000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
bash 
# Start with Docker Compose
docker-compose up -d

# View logs
docker-compose logs -f mynadb

# Stop
docker-compose down

Build Docker Image Locally

dockerfile 
# Dockerfile
FROM rust:1.75-slim as builder

WORKDIR /app
COPY . .
RUN cargo build --release

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*

COPY --from=builder /app/target/release/mynadb /usr/local/bin/

EXPOSE 7000 7001
VOLUME ["/data"]

CMD ["mynadb"]
bash 
# Build the image
docker build -t mynadb:local .

# Run locally built image
docker run -d -p 7000:7000 -p 7001:7001 mynadb:local

Systemd Service

Set up MynaDB as a systemd service for automatic startup and management.

Create Service User

bash 
# Create a dedicated user
sudo useradd --system --no-create-home --shell /bin/false mynadb

# Create data directory
sudo mkdir -p /var/lib/mynadb
sudo chown mynadb:mynadb /var/lib/mynadb

# Create config directory
sudo mkdir -p /etc/mynadb

Create Configuration File

toml 
# /etc/mynadb/config.toml

[server]
http_port = 7000
flight_port = 7001

[storage]
data_dir = "/var/lib/mynadb/data"

[cache]
flush_interval_secs = 30

[compaction]
min_files = 4
max_level = 5
max_concurrent_compactions = 8

[retention]
enabled = true
default_days = 30
interval_secs = 3600

Create Systemd Unit File

ini 
# /etc/systemd/system/mynadb.service

[Unit]
Description=MynaDB - High-Performance Time Series Database
Documentation=https://github.com/mynadb/mynadb
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=mynadb
Group=mynadb
ExecStart=/usr/local/bin/mynadb --config /etc/mynadb/config.toml
Restart=on-failure
RestartSec=5

# Resource limits
LimitNOFILE=65536
LimitNPROC=4096

# Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/lib/mynadb
PrivateTmp=true

# Environment
Environment=RUST_LOG=info

[Install]
WantedBy=multi-user.target

Enable and Start Service

bash 
# Reload systemd
sudo systemctl daemon-reload

# Enable service to start on boot
sudo systemctl enable mynadb

# Start the service
sudo systemctl start mynadb

# Check status
sudo systemctl status mynadb

# View logs
sudo journalctl -u mynadb -f

Verify Installation

After installation, verify MynaDB is running correctly:

Health Check

bash 
# Check HTTP health endpoint
curl http://localhost:7000/health

# Expected response:
{"status": "ok"}

Write Test Data

bash 
# Write test data
curl -X POST 'http://localhost:7000/write/testdb' \
  -d 'cpu,host=server01 usage=45.2'

# Flush to disk
curl -X POST http://localhost:7000/flush

# Query data
curl 'http://localhost:7000/query/testdb' \
  -d 'SELECT * FROM cpu'

Check Ports

bash 
# Verify ports are listening
ss -tlnp | grep -E '7000|7001'

# Expected output:
LISTEN  0  128  *:7000  *:*  users:(("mynadb",pid=1234,fd=3))
LISTEN  0  128  *:7001  *:*  users:(("mynadb",pid=1234,fd=4))
Installation Complete!

MynaDB is now installed and running. Continue to the Quick Start guide to learn how to use it, or see Configuration for advanced setup options.

Troubleshooting

Issue Solution
Port already in use Change http_port or flight_port in config.toml
Permission denied Check data directory ownership: chown -R mynadb:mynadb /var/lib/mynadb
Service won't start Check logs: journalctl -u mynadb -n 50
Out of memory Reduce full_compaction_max_concurrent in config
Slow queries Trigger compaction: curl -X POST http://localhost:7000/compact/mydb
