Arrow Flight API

MynaDB 提供 Apache Arrow Flight 接口，用于高性能数据流式传输。 Arrow Flight 是一种基于 gRPC 的协议，针对大规模数据集的零拷贝传输进行了优化。

✈ 为什么选择 Arrow Flight？

对于大型结果集，Arrow Flight 的数据传输速度比 REST API 快 10-100 倍。 它非常适合分析仪表盘、数据管道和机器学习工作负载。

概述

Arrow Flight API 提供以下功能：

  • 高吞吐量流式传输 - 每秒传输数百万行数据
  • 零拷贝传输 - 数据保持 Arrow 列式格式
  • SQL 查询执行 - 完整的 DataFusion SQL 支持
  • 并行数据检索 - 多端点分布式读取
  • 压缩 - 内置 LZ4/ZSTD 压缩
特性 HTTP API Arrow Flight
协议 REST/JSON gRPC/Arrow IPC
数据格式 JSON（文本） Arrow（二进制）
适用场景 小型查询、写入 大型结果集
流式传输 否 是
客户端库 任何 HTTP 客户端 Arrow Flight 客户端

连接

Arrow Flight 服务器默认运行在端口 7001（可通过 flight_port 配置）。

默认端口 7001
协议 gRPC (HTTP/2)
连接字符串 grpc://localhost:7001

配置

toml 
[server]
http_port = 7000
flight_port = 7001  # Arrow Flight gRPC port

DataFusion CLI

通过 Flight 查询 MynaDB 最简单的方式是使用 DataFusion CLI。 它提供了支持 Arrow Flight 的交互式 SQL 命令行。

安装

bash 
# Install via Cargo
cargo install datafusion-cli

# Or download pre-built binary
# https://github.com/apache/arrow-datafusion/releases

连接到 MynaDB

bash 
# Connect to Flight server
datafusion-cli --host localhost --port 7001

# With specific database catalog
datafusion-cli --host localhost --port 7001 --catalog mydb

交互式会话

sql 
-- List available tables
SHOW TABLES;

-- Query data
SELECT host, AVG(usage) AS avg_cpu
FROM mydb.default.cpu
WHERE time > now() - interval '1 hour'
GROUP BY host
ORDER BY avg_cpu DESC;

-- Describe table schema
DESCRIBE mydb.default.cpu;

Python 客户端

使用 pyarrow 通过 Arrow Flight 连接 MynaDB。

安装

bash 
pip install pyarrow pandas

基本查询

python 
import pyarrow.flight as flight
import pandas as pd

# Connect to MynaDB
client = flight.connect("grpc://localhost:7001")

# Execute SQL query
sql = """
    SELECT host, region, AVG(usage) as avg_cpu
    FROM mydb.default.cpu
    WHERE time > now() - interval '1 hour'
    GROUP BY host, region
"""

# Get Flight info and read data
flight_info = client.get_flight_info(flight.FlightDescriptor.for_command(sql))
reader = client.do_get(flight_info.endpoints[0].ticket)

# Convert to Pandas DataFrame
df = reader.read_pandas()
print(df)

流式读取大型结果集

python 
import pyarrow.flight as flight

client = flight.connect("grpc://localhost:7001")

sql = "SELECT * FROM mydb.default.cpu"
flight_info = client.get_flight_info(flight.FlightDescriptor.for_command(sql))

# Stream record batches (memory efficient)
reader = client.do_get(flight_info.endpoints[0].ticket)

total_rows = 0
for batch in reader:
    total_rows += batch.data.num_rows
    # Process each batch...
    print(f"Received batch with {batch.data.num_rows} rows")

print(f"Total rows: {total_rows}")

与 Pandas/Polars 集成

python 
import pyarrow.flight as flight
import polars as pl

def query_stats_service(sql: str) -> pl.DataFrame:
    """Execute SQL and return Polars DataFrame."""
    client = flight.connect("grpc://localhost:7001")
    flight_info = client.get_flight_info(
        flight.FlightDescriptor.for_command(sql)
    )
    reader = client.do_get(flight_info.endpoints[0].ticket)

    # Read as Arrow table, convert to Polars
    arrow_table = reader.read_all()
    return pl.from_arrow(arrow_table)

# Example usage
df = query_stats_service("""
    SELECT
        date_trunc('minute', to_timestamp(timestamp/1000000000)) as minute,
        host,
        AVG(usage) as avg_cpu,
        MAX(usage) as max_cpu
    FROM mydb.default.cpu
    GROUP BY 1, 2
    ORDER BY 1 DESC
    LIMIT 100
""")
print(df)

Rust 客户端

使用 arrow-flight crate 从 Rust 应用程序进行连接。

Cargo.toml

toml 
[dependencies]
arrow-flight = "50"
arrow = "50"
tonic = "0.10"
tokio = { version = "1", features = ["full"] }
futures = "0.3"

示例客户端

rust 
use arrow_flight::flight_service_client::FlightServiceClient;
use arrow_flight::{FlightDescriptor, Ticket};
use futures::StreamExt;

async fn query_stats_service() -> Result<(), Box<dyn std::error::Error>> {
    // Connect to Flight server
    let mut client = FlightServiceClient::connect(
        "http://localhost:7001"
    ).await?;

    // Create SQL command
    let sql = "SELECT * FROM mydb.default.cpu LIMIT 100";
    let descriptor = FlightDescriptor::new_cmd(sql.to_string());

    // Get flight info
    let flight_info = client
        .get_flight_info(descriptor)
        .await?
        .into_inner();

    // Read data from first endpoint
    if let Some(endpoint) = flight_info.endpoint.first() {
        let ticket = endpoint.ticket.clone().unwrap();
        let mut stream = client.do_get(ticket).await?.into_inner();

        while let Some(batch) = stream.next().await {
            let batch = batch?;
            println!("Received batch with {} bytes", batch.data_body.len());
        }
    }

    Ok(())
}

通过 Flight 执行 SQL 查询

MynaDB 中的 Arrow Flight 使用 FlightSQL 语义。 SQL 查询以命令描述符的形式传递。

支持的操作

操作 说明 示例
SELECT 查询数据 SELECT * FROM cpu
SHOW TABLES 列出表 SHOW TABLES
DESCRIBE 表 Schema DESCRIBE cpu
聚合 GROUP BY、窗口函数 SELECT AVG(x) FROM t GROUP BY y
连接查询 跨表查询 SELECT * FROM a JOIN b ON ...

表命名

-- Fully qualified name (recommended)
SELECT * FROM mydb.default.cpu;

-- With schema
SELECT * FROM default.cpu;

-- Simple name (uses default catalog/schema)
SELECT * FROM cpu;

性能建议

🔥 性能最佳实践
  • 使用列投影 - 只选择需要的列
  • 应用时间过滤 - 利用分区裁剪
  • 对大数据集使用流式读取
  • 启用压缩以优化网络传输

列投影

-- Good: Only select needed columns
SELECT time, host, usage FROM cpu;

-- Avoid: Selecting all columns
SELECT * FROM cpu;

时间范围过滤

-- Leverages date partition pruning
SELECT * FROM cpu
WHERE time >= 1704067200000000000  -- epoch nanos
  AND time <  1704153600000000000;

-- Or using date functions
SELECT * FROM cpu
WHERE time > now() - interval '1 day';

压缩选项

python 
import pyarrow.flight as flight

# Enable gRPC compression
options = flight.FlightCallOptions(
    headers=[(b"grpc-encoding", b"gzip")]
)

client = flight.connect("grpc://localhost:7001")
flight_info = client.get_flight_info(
    flight.FlightDescriptor.for_command(sql),
    options=options
)

故障排除

问题 解决方案
连接被拒绝 检查 flight_port 配置，确保服务器正在运行
找不到表 使用完全限定名：catalog.schema.table
大型查询超时 增加 gRPC 超时时间，使用流式读取
内存问题 使用流式批次读取，避免使用 read_all()
