安装

本指南介绍安装 MynaDB 的多种方式：预编译二进制文件、从源码构建、Docker 容器，以及设置为系统服务。

系统要求

硬件

组件 最低要求 推荐配置
CPU 2 核 4+ 核
内存 2 GB 8+ GB
存储 10 GB SSD 100+ GB NVMe SSD
网络 100 Mbps 1 Gbps+

软件

  • 操作系统：Linux（Ubuntu 20.04+、Debian 11+、RHEL 8+ 等）
  • 架构：x86_64 (AMD64) 或 ARM64
  • Rust：1.75+（从源码构建时需要）

二进制发行版

安装 MynaDB 最简单的方式是使用预编译的二进制文件。

下载最新版本

bash 
# 下载最新版本
curl -LO https://github.com/mynadb/mynadb/releases/latest/download/mynadb-linux-amd64.tar.gz

# 解压归档文件
tar -xzf mynadb-linux-amd64.tar.gz

# 移动二进制文件到 /usr/local/bin
sudo mv mynadb /usr/local/bin/

# 验证安装
mynadb --version

ARM64（Apple Silicon、AWS Graviton）

bash 
# 下载 ARM64 版本
curl -LO https://github.com/mynadb/mynadb/releases/latest/download/mynadb-linux-arm64.tar.gz

# 解压并安装
tar -xzf mynadb-linux-arm64.tar.gz
sudo mv mynadb /usr/local/bin/

从源码构建

从源码构建可以获取最新功能并支持自定义。

前置条件

bash 
# 安装 Rust（如果尚未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 重新加载 shell 配置
source $HOME/.cargo/env

# 验证 Rust 安装
rustc --version  # 应为 1.75+

克隆并构建

bash 
# 克隆仓库
git clone https://github.com/mynadb/mynadb.git
cd mynadb

# 构建 release 二进制文件
cargo build --release

# 二进制文件位于 target/release/mynadb
./target/release/mynadb --version

# 可选：安装到系统路径
sudo cp target/release/mynadb /usr/local/bin/

构建脚本

使用提供的构建脚本创建可分发的归档文件：

bash 
# 构建并创建发行归档文件
./build.sh

# 输出：mynadb-vX.X.X-linux-amd64.tar.gz

Docker

在 Docker 容器中运行 MynaDB，便于部署和隔离。

拉取官方镜像

bash 
# 拉取最新镜像
docker pull ghcr.io/mynadb/mynadb:latest

# 使用默认设置运行
docker run -d \
  --name mynadb \
  -p 7000:7000 \
  -p 7001:7001 \
  -v stats-data:/data \
  ghcr.io/mynadb/mynadb:latest

Docker Compose

生产环境部署建议使用 Docker Compose：

yaml 
# docker-compose.yml
version: '3.8'

services:
  mynadb:
    image: ghcr.io/mynadb/mynadb:latest
    container_name: mynadb
    restart: unless-stopped
    ports:
      - "7000:7000"   # HTTP API
      - "7001:7001"   # Arrow Flight
    volumes:
      - ./data:/data
      - ./config.toml:/etc/mynadb/config.toml:ro
    environment:
      - RUST_LOG=info
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:7000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
bash 
# 使用 Docker Compose 启动
docker-compose up -d

# 查看日志
docker-compose logs -f mynadb

# 停止
docker-compose down

本地构建 Docker 镜像

dockerfile 
# Dockerfile
FROM rust:1.75-slim as builder

WORKDIR /app
COPY . .
RUN cargo build --release

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*

COPY --from=builder /app/target/release/mynadb /usr/local/bin/

EXPOSE 7000 7001
VOLUME ["/data"]

CMD ["mynadb"]
bash 
# 构建镜像
docker build -t mynadb:local .

# 运行本地构建的镜像
docker run -d -p 7000:7000 -p 7001:7001 mynadb:local

Systemd 服务

将 MynaDB 设置为 systemd 服务，实现自动启动和管理。

创建服务用户

bash 
# 创建专用用户
sudo useradd --system --no-create-home --shell /bin/false mynadb

# 创建数据目录
sudo mkdir -p /var/lib/mynadb
sudo chown mynadb:mynadb /var/lib/mynadb

# 创建配置目录
sudo mkdir -p /etc/mynadb

创建配置文件

toml 
# /etc/mynadb/config.toml

[server]
http_port = 7000
flight_port = 7001

[storage]
data_dir = "/var/lib/mynadb/data"

[cache]
flush_interval_secs = 30

[compaction]
min_files = 4
max_level = 5
max_concurrent_compactions = 8

[retention]
enabled = true
default_days = 30
interval_secs = 3600

创建 Systemd 单元文件

ini 
# /etc/systemd/system/mynadb.service

[Unit]
Description=MynaDB - High-Performance Time Series Database
Documentation=https://github.com/mynadb/mynadb
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=mynadb
Group=mynadb
ExecStart=/usr/local/bin/mynadb --config /etc/mynadb/config.toml
Restart=on-failure
RestartSec=5

# 资源限制
LimitNOFILE=65536
LimitNPROC=4096

# 安全加固
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/lib/mynadb
PrivateTmp=true

# 环境变量
Environment=RUST_LOG=info

[Install]
WantedBy=multi-user.target

启用并启动服务

bash 
# 重新加载 systemd
sudo systemctl daemon-reload

# 设置开机自启
sudo systemctl enable mynadb

# 启动服务
sudo systemctl start mynadb

# 检查状态
sudo systemctl status mynadb

# 查看日志
sudo journalctl -u mynadb -f

验证安装

安装完成后，验证 MynaDB 是否正常运行：

健康检查

bash 
# 检查 HTTP 健康端点
curl http://localhost:7000/health

# 预期响应：
{"status": "ok"}

写入测试数据

bash 
# 写入测试数据
curl -X POST 'http://localhost:7000/write/testdb' \
  -d 'cpu,host=server01 usage=45.2'

# 刷盘到磁盘
curl -X POST http://localhost:7000/flush

# 查询数据
curl 'http://localhost:7000/query/testdb' \
  -d 'SELECT * FROM cpu'

检查端口

bash 
# 验证端口是否在监听
ss -tlnp | grep -E '7000|7001'

# 预期输出：
LISTEN  0  128  *:7000  *:*  users:(("mynadb",pid=1234,fd=3))
LISTEN  0  128  *:7001  *:*  users:(("mynadb",pid=1234,fd=4))
安装完成！

MynaDB 已安装并正在运行。请继续阅读快速入门指南了解如何使用， 或查看配置了解高级设置选项。

故障排除

问题 解决方案
端口已被占用 在 config.toml 中修改 http_port 或 flight_port
权限被拒绝 检查数据目录所有权：chown -R mynadb:mynadb /var/lib/mynadb
服务无法启动 检查日志：journalctl -u mynadb -n 50
内存不足 在配置中减小 full_compaction_max_concurrent 的值
查询缓慢 触发压缩合并：curl -X POST http://localhost:7000/compact/mydb
