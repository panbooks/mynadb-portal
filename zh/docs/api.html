HTTP API 参考

向您的数据提问。MynaDB 在端口 7000（可配置）上公开 REST API，让您与基础设施数据进行交互。除非另有说明，所有端点都接受并返回 JSON。

💡 基础 URL

http://localhost:7000（默认配置）

写入数据

使用 InfluxDB Line Protocol 写入时序数据。

端点 POST /write/:db
Content-Type text/plain
参数 :db - 数据库名称（不存在时自动创建）

Line Protocol 格式

<measurement>,<tag_key>=<tag_value>,... <field_key>=<field_value>,... [timestamp]

示例

bash 
curl -X POST http://localhost:7000/write/mydb \
  -H "Content-Type: text/plain" \
  -d 'cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-east usage=82.3,temp=50
memory,host=server01 used=16.5,total=32.0'

响应

HTTP/1.1 204 No Content

字段类型

后缀 类型 示例
（无） Float64 usage=75.5
i Int64 count=42i
u UInt64 bytes=1024u
引号 String status="ok"
t/f Boolean active=t

查询数据

对时序数据执行 SQL 查询。

端点 GET /query/:db?sql=<query>
POST /query/:db
Content-Type text/plain（POST 请求体为 SQL）
响应 JSON 对象数组

GET 请求

bash 
curl 'http://localhost:7000/query/mydb?sql=SELECT * FROM cpu LIMIT 10'

POST 请求（推荐用于复杂查询）

bash 
curl -X POST http://localhost:7000/query/mydb \
  -d "SELECT host, AVG(usage) as avg_cpu
      FROM cpu
      WHERE region = 'us-west'
      GROUP BY host
      ORDER BY avg_cpu DESC"

响应

[
  {"host": "server01", "avg_cpu": 75.5},
  {"host": "server02", "avg_cpu": 45.2}
]

表名格式

格式 示例 说明
短名称 SELECT * FROM cpu 自动添加 catalog.schema 前缀
Schema.table SELECT * FROM default.cpu 显式指定 schema
完全限定 SELECT * FROM mydb.default.cpu 完整的 catalog.schema.table

刷新缓存

将内存缓存刷新为磁盘上的 Parquet 文件。

端点 POST /flush
bash 
curl -X POST http://localhost:7000/flush

响应

{"status": "ok", "message": "Cache flushed successfully"}

触发压缩合并

手动触发数据库或特定 measurement 的压缩合并。

端点 POST /compact/:db
POST /compact/:db/:measurement
bash 
# 压缩整个数据库
curl -X POST http://localhost:7000/compact/mydb

# 压缩特定 measurement
curl -X POST http://localhost:7000/compact/mydb/cpu

列出表

列出数据库中的所有表。

端点 GET /tables/:db
bash 
curl http://localhost:7000/tables/mydb

响应

{
  "catalog": "mydb",
  "tables": [
    "mydb.default.cpu",
    "mydb.default.memory",
    "mydb.default.disk"
  ],
  "count": 3
}

获取表结构

获取表的结构（列和类型）。

端点 GET /schema/:db/:table
bash 
curl http://localhost:7000/schema/mydb/cpu

响应

{
  "table": "cpu",
  "columns": [
    {"name": "timestamp", "type": "Int64"},
    {"name": "host", "type": "Utf8"},
    {"name": "region", "type": "Utf8"},
    {"name": "usage", "type": "Float64"},
    {"name": "temp", "type": "Int64"}
  ]
}

列出分区

列出表的日期分区。

端点 GET /partitions/:catalog/:schema/:table
bash 
curl http://localhost:7000/partitions/mydb/default/cpu

删除分区

curl -X DELETE http://localhost:7000/partitions/mydb/default/cpu/2024-01-15

数据保留

根据时间或日期删除旧分区。

端点 POST /retention/:catalog/:schema/:table
Content-Type application/json

按时间删除

bash 
curl -X POST http://localhost:7000/retention/mydb/default/cpu \
  -H "Content-Type: application/json" \
  -d '{"older_than_days": 30}'

按日期删除

curl -X POST http://localhost:7000/retention/mydb/default/cpu \
  -H "Content-Type: application/json" \
  -d '{"before_date": "2024-01-01"}'

健康检查

检查服务器是否正在运行。

端点 GET /health 
curl http://localhost:7000/health

响应

{"status": "ok"}

缓存统计

获取内存缓存的统计信息。

端点 GET /stats 
curl http://localhost:7000/stats

OpenTelemetry (OTLP) 数据接入

原生 OpenTelemetry 数据接入，支持指标、日志和链路追踪。参见 完整 OTLP 文档。

端点 方法 说明
gRPC :4317 - OTLP gRPC 接收器（指标、日志、链路追踪）
/v1/metrics POST OTLP HTTP 指标接收器
/v1/logs POST OTLP HTTP 日志接收器
/v1/traces POST OTLP HTTP 链路追踪接收器

Iceberg REST Catalog

Apache Iceberg REST Catalog API，支持多引擎分析。参见 完整 Iceberg 文档。

端点 方法 说明
/iceberg/v1/namespaces GET 列出命名空间
/iceberg/v1/namespaces/:ns/tables GET 列出命名空间中的表
/iceberg/v1/namespaces/:ns/tables/:table GET 获取表元数据

Grafana 兼容端点

InfluxDB 兼容端点，用于 Grafana 数据源集成。参见 完整 Grafana 文档。

端点 方法 说明
/api/v2/query POST Flux 兼容查询端点
/api/v1/query GET/POST InfluxQL 兼容查询端点

流式查询

实时查询订阅。参见 完整流式查询文档。

端点 方法 说明
/ws/query/:db WebSocket WebSocket 查询订阅
/stream/query/:db GET Server-Sent Events 查询流

Prometheus 指标

Prometheus 兼容的指标端点。参见 完整监控文档。

端点 GET /metrics 
curl http://localhost:7000/metrics

认证

启用认证时的 API 认证。参见 完整安全文档。

端点 方法 说明
/auth/users GET 列出用户
/auth/users POST 创建用户
/auth/users/:name DELETE 删除用户
/auth/users/:name/role PUT 更新用户角色

错误响应

所有错误都返回包含错误信息的 JSON：

{
  "error": "Table not found: mydb.default.unknown"
}
状态码 说明
200 成功（有响应体）
204 成功（无内容）
400 请求错误（语法无效）
404 未找到（表、数据库）
500 服务器内部错误
