集群运维指南

通过管理控制台管理 MynaDB 集群的详细说明。本指南涵盖节点管理、表操作、缩容工作流、事件监控、Meta 和数据操作、调试工具以及设置。

节点管理

节点页面（/nodes）以卡片网格布局显示所有已注册的集群节点。每张卡片显示节点 ID、广播地址以及角色、状态和模式的彩色标记。

过滤器

使用顶部的过滤栏缩小节点列表范围：

过滤器 选项 说明
搜索 自由文本 按 ID 或地址过滤节点
角色 Meta、Router、Data、Compactor 按节点的集群角色过滤
状态 活跃、非活跃、排空中、退役中、已排空 按运行状态过滤
模式 主节点、备用节点 按节点模式过滤（仅限数据节点）

节点卡片

每张节点卡片显示以下信息：

  • 节点 ID -- 注册时分配的唯一标识符
  • 广播地址 -- 其他节点用来访问此节点的 host:port
  • 角色标记 -- Meta（蓝色）、Router（绿色）、Data（紫色）、Compactor（橙色）
  • 状态标记 -- 活跃（绿色）、非活跃（灰色）、排空中（黄色）、退役中（橙色）、已排空（红色）
  • 模式标记 -- 主节点或备用节点（仅限数据节点）
  • 可用区 -- 可用区或机架分配
  • 拥有的表 -- 分配给此节点的表数量
  • 最后心跳 -- 最后一次健康检查的相对时间戳

点击任意节点卡片可导航到节点详情页面。

节点详情页面

节点详情页面（/nodes/:id）提供单个节点的完整信息和管理操作：

操作 说明
刷新节点 强制节点将内存中的写入缓存刷新到 Parquet 文件
压缩合并节点 触发此节点上所有表的压缩合并
清除节点缓存 清除节点的元数据和查询结果缓存
退役节点 开始退役节点——将表迁移到其他节点，然后排空
取消退役 取消正在进行的退役操作（在排空/退役状态下可用）

当节点处于退役中或排空中状态时，详情页面会显示进度跟踪器，包含已迁移和剩余的表数量。

⚠ 退役与移除

始终使用退役操作而不是突然停止数据节点。退役操作会在节点关闭前将表所有权优雅地迁移到其他健康节点，避免数据不可用。

表管理

表页面（/tables）列出集群中所有已注册的表及其关键元数据和管理操作。

表列表

表列表支持搜索、按 catalog 和 schema 过滤，以及分页（每页 20 个表）。每行显示：

列 说明
名称 完全限定名（catalog.schema.table）
大小 所有分区的总存储大小
行数 近似总行数
分区数 日期分区数量
所有者节点 负责此表的主数据节点（可点击链接到节点详情）
副本数 跨节点的副本数量
操作 重新分配按钮，将表移动到不同节点

重新分配表

点击任意表行上的重新分配按钮，打开一个对话框选择目标数据节点。表重新分配会转移所有权，使新节点处理该表的写入和压缩合并。

重新分配表的常见原因：

  • 负载均衡 -- 将表从过载节点移至较空闲的节点
  • 维护 -- 在节点维护前将表迁移出去
  • 扩容 -- 添加新数据节点后重新分配表
  • 恢复 -- 将故障节点的表转移到健康节点

缩容操作

缩容页面（/scale-in）提供引导式三步向导，用于优雅地减少集群中的数据节点数量。

步骤 1：配置

在配置步骤中，您需要指定：

  • 节点数量 -- 要从集群中移除的节点数量
  • 策略 -- 选择要退役节点的算法：
策略 说明
最少负载 退役拥有最少表的节点（最小化迁移工作量）
先进先出 按注册时间退役最早的节点
后进先出 退役最近添加的节点
高可用单元 从同一可用区退役节点以维持区域均衡

您也可以手动选择特定节点 ID 而不使用自动策略。向导会根据当前集群状态显示推荐方案。

步骤 2：预览

配置完成后，向导执行试运行并显示缩容计划：

  • 计划摘要 -- 要退役的总节点数、要迁移的表数、预估时间
  • 待退役节点 -- 已选择的节点列表及其当前表数量
  • 执行阶段 -- 有序的迁移步骤列表，显示哪些表移动到哪里

在继续之前请仔细审查计划。您可以返回步骤 1 调整配置。

步骤 3：执行

点击执行开始缩容操作。页面显示实时进度：

  • 进度条 -- 总体完成百分比
  • 阶段状态 -- 当前阶段及其状态（等待中、进行中、已完成）
  • 每节点进度 -- 每个退役节点的已迁移表数与总数
  • 操作状态 -- 运行中、已完成或失败（含错误详情）

您可以取消正在运行的缩容操作。取消会停止后续迁移，但不会回滚已完成的表迁移。

💡 缩容最佳实践

建议每次缩减一到两个节点，而不是一次缩减多个。这可以限制正在进行的表迁移数量，减少对查询性能的影响。详情请参阅集群扩缩容的架构说明。

事件监控

事件页面（/events）显示通过 WebSocket 传输的集群事件实时流。

实时流

事件以逆时间顺序以卡片形式出现。每张事件卡片显示：

  • 时间戳 -- 相对时间（如"2 分钟前"），悬停时显示完整 ISO 时间戳
  • 类型图标 -- 表示事件类别的彩色图标
  • 来源 -- 生成事件的节点或组件
  • 严重性 -- 信息、警告或错误事件的视觉指示器
  • 详情 -- 事件特定的数据载荷

连接状态

状态指示器显示与管理节点的 WebSocket 连接是否处于活跃状态。如果连接断开，页面会自动尝试重新连接。使用暂停/恢复可以在不断开连接的情况下暂时停止实时流。

过滤器

过滤器 选项
时间范围 全部时间、最近 1 小时、最近 24 小时、最近 7 天
事件类型 从下列事件类型中选择
事件来源 按来源节点或组件过滤

事件类型

事件类型 说明
NodeJoined 节点向 Meta 服务注册
NodeLeft 节点注销或超时
NodeStateChanged 节点在不同状态之间转换（活跃、退役中、已排空等）
LeaderChanged 选举了新的 Meta Leader
TableReassigned 表的所有权被移动到不同节点
TableOwnerSet 首次为表分配所有者
TableOwnerRemoved 表的所有者被移除（节点故障或注销）
CacheInvalidate 向数据节点广播了缓存失效消息
SchemaChanged 表的 schema 被修改（新列、类型变更）
FlushForRecovery 指示节点刷新缓存以进行 WAL 恢复
DistributionConfigChanged 表分布配置已更新
TablesCacheRefresh 一个或多个节点上的表缓存已刷新
Unknown 未识别的事件类型（前向兼容）

导出

使用导出按钮下载当前事件列表：

  • JSON -- 包含所有元数据的完整事件对象
  • CSV -- 表格格式，适合电子表格分析

启用浏览器通知（铃铛图标）以接收节点故障或 Leader 变更等重要事件的桌面提醒。

Meta 操作

Meta 操作页面（/meta-operations）提供对 Meta 服务集群的可视化和控制。

Meta 集群状态

状态区域显示：

  • 集群健康 -- 如果大多数 Meta 节点可达则为健康
  • 当前任期 -- Raft 共识任期编号
  • 提交索引 -- 最新已提交的日志条目索引
  • Meta Leader -- 当前 Leader 的节点 ID 和地址

Meta 节点

表格列出所有 Meta 服务实例及其地址、角色（Leader/Follower）和连接状态。

Leader 选举

两个按钮允许您手动触发 Leader 选举操作：

  • 竞选 -- 强制当前 Meta Leader 发起新的选举
  • 辞任 -- 要求当前 Leader 下台，触发新的选举
⚠ 选举注意事项

手动触发选举会在新 Leader 选举期间造成短暂中断。仅在维护窗口期间或排查 Leader 问题时使用这些操作。

分布配置

分布配置控制新表如何分配到数据节点。您可以查看和修改：

  • 范围 -- 在 catalog、schema 或节点级别应用分布规则
  • 复制因子 -- 表数据的副本数量
  • 放置策略 -- 选择哪些节点接收新表的算法

数据操作

数据操作页面（/data-operations）提供集群中所有数据节点的批量和单节点操作。

汇总卡片

顶部的汇总卡片显示数据节点总数、活跃节点数和最后一次操作状态。

批量操作

操作 说明
全部刷新 同时在所有活跃数据节点上触发缓存刷新
全部压缩合并 同时在所有活跃数据节点上触发压缩合并

单节点操作

批量操作下方的表格列出每个数据节点及其单独的操作按钮：

操作 说明
刷新 将节点的内存写入缓存刷新到磁盘/对象存储的 Parquet 文件
压缩合并 对该节点拥有的所有表运行压缩合并，将小的 L0 文件合并为更大的 L1/L2 文件
清除缓存 清除节点的元数据缓存和查询结果缓存以释放内存

每个操作按钮在操作进行时显示加载动画。每行上的状态标记指示节点当前的运行状态。

调试工具

调试工具页面（/debug）提供测试、性能分析和缓存管理的实用工具。

测试数据生成

生成用于测试和演示的示例时序数据。可配置参数：

选项 说明
数据类型 CPU、内存、磁盘、网络指标（选择一个或多个）
数据库 生成数据的目标 catalog 名称
日期范围 生成时序数据的起止日期
数据点数 每种指标类型生成的数据点数量

点击生成通过数据节点的写入 API 写入测试数据。数据可立即在查询控制台中查询。

性能分析

从数据节点下载性能分析快照：

  • 堆内存分析 -- 显示内存分配情况的快照，了解哪些对象占用了内存
  • CPU 分析 -- CPU 使用率分析，用于识别热点代码路径

分析结果以文件形式下载，可使用 pprof 或火焰图生成器等标准工具进行分析。

缓存管理

  • 预热缓存 -- 将特定 catalog 的元数据主动加载到数据节点缓存中，降低首次查询延迟
  • 清除所有缓存 -- 清除所有数据节点上的缓存元数据和查询结果以释放内存

设置

设置页面（/settings）以只读视图显示当前管理节点的配置。

显示信息

  • 管理配置 -- HTTP 主机、端口、Meta 端点
  • 集群信息 -- 集群 ID、部署模式
  • 网络 -- 绑定地址和连接详情
  • 节点数量 -- 已注册的 Meta、Router 和数据节点汇总

使用刷新按钮从管理节点重新加载配置。

💡 配置变更

设置页面为只读。要更改管理节点配置，请编辑 TOML 配置文件并重启管理节点。可用选项请参阅管理指南 - 配置。
