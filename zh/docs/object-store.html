对象存储支持

MynaDB 支持云对象存储后端，包括 AWS S3、MinIO 和阿里云 OSS。这使得大规模时序数据集的存储具备可扩展性和成本效益。

配置

本地存储（默认）

toml 
[storage]
# Local filesystem storage
backend = "local"
data_dir = "/data/mynadb"

AWS S3

toml 
[storage]
backend = "s3"
data_dir = "/data/mynadb"  # Local cache/WAL directory

[storage.s3]
bucket = "my-mynadb-bucket"
region = "us-west-2"

# Credentials (optional if using IAM roles)
access_key = "AKIAIOSFODNN7EXAMPLE"
secret_key = "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"

# Optional: custom endpoint for S3-compatible services
# endpoint = "https://s3.us-west-2.amazonaws.com"

# Optional: path prefix within bucket
prefix = "mynadb-data"

MinIO

toml 
[storage]
backend = "minio"
data_dir = "/data/mynadb"

[storage.s3]
bucket = "mynadb"
region = "us-east-1"  # Can be any value for MinIO

# MinIO credentials
access_key = "minioadmin"
secret_key = "minioadmin"

# MinIO endpoint
endpoint = "http://localhost:9000"

# Allow HTTP (required for local MinIO)
allow_http = true

阿里云 OSS

toml 
[storage]
backend = "oss"
data_dir = "/data/mynadb"

[storage.s3]
bucket = "my-oss-bucket"
region = "oss-cn-hangzhou"

# Aliyun credentials
access_key = "your-access-key-id"
secret_key = "your-access-key-secret"

# OSS endpoint
endpoint = "https://oss-cn-hangzhou.aliyuncs.com"

搭建 MinIO（本地开发）

Docker 部署

bash 
# Run MinIO with Docker
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v minio-data:/data \
  minio/minio server /data --console-address ":9001"

# Create a bucket using MinIO client
docker exec minio mc alias set local http://localhost:9000 minioadmin minioadmin
docker exec minio mc mb local/mynadb

Docker Compose

yaml 
version: '3.8'
services:
  minio:
    image: minio/minio
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    command: server /data --console-address ":9001"
    volumes:
      - minio-data:/data

  mynadb:
    image: mynadb:latest
    ports:
      - "7000:7000"
      - "7001:7001"
    volumes:
      - ./config.toml:/etc/mynadb/config.toml
    depends_on:
      - minio

volumes:
  minio-data:

IAM 角色认证（AWS）

在 AWS 上运行时（EC2、ECS、EKS），可以使用 IAM 角色代替访问密钥：

toml 
[storage]
backend = "s3"
data_dir = "/data/mynadb"

[storage.s3]
bucket = "my-mynadb-bucket"
region = "us-west-2"
# No access_key or secret_key - uses IAM role

所需 IAM 策略

json 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-mynadb-bucket",
        "arn:aws:s3:::my-mynadb-bucket/*"
      ]
    }
  ]
}

对象存储文件缓存

使用对象存储时，建议启用文件缓存以降低延迟：

toml 
[file_cache]
# Enable file caching (highly recommended for Object Store)
enabled = true

# Local SSD cache directory
cache_dir = "/var/cache/mynadb/parquet"

# Maximum cache size (100 GB)
max_size_bytes = 107374182400

# Save cache index every 5 minutes
auto_save_interval_secs = 300

性能注意事项

网络延迟

  • 对象存储操作比本地磁盘延迟更高
  • 启用文件缓存以缓解读取延迟
  • 将 MynaDB 实例部署在与存储桶同一区域

写入性能

  • 写入操作在 DataCache（内存）中缓冲
  • 后台刷写异步写入对象存储
  • 增大 flush_interval_secs 以获得更好的批量效果

读取性能

  • 对频繁访问的数据使用文件缓存
  • 针对可预测的查询模式预热缓存
  • 利用日期过滤器进行分区裁剪

数据迁移

本地迁移到 S3

bash 
# 1. Stop MynaDB
systemctl stop mynadb

# 2. Sync data to S3
aws s3 sync /data/mynadb s3://my-bucket/mynadb-data/

# 3. Update config.toml to use S3
# 4. Start MynaDB
systemctl start mynadb

S3 迁移到本地

bash 
# 1. Stop MynaDB
systemctl stop mynadb

# 2. Sync data from S3
aws s3 sync s3://my-bucket/mynadb-data/ /data/mynadb

# 3. Update config.toml to use local storage
# 4. Start MynaDB
systemctl start mynadb
