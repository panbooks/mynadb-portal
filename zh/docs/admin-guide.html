管理仪表盘指南

MynaDB 管理控制台的全面指南——从集群健康监控到交互式 SQL 查询。了解仪表盘的工作原理、每个页面的功能，以及如何通过浏览器开始管理您的集群。

概述

MynaDB 管理控制台是一个独立的 React Web 应用程序，提供对 MynaDB 集群的全面可视化和控制。它作为单独的管理节点运行，连接到 Meta 服务进行集群发现，并将请求转发到相应的数据节点。

架构

管理节点位于集群旁边，与两个层面进行通信：

text 
  Browser (Admin UI)
        |
        v
  +------------------+
  |   Admin Node     |  Port 18080
  |   (Axum HTTP)    |
  +--------+---------+
           |
     +-----+-----+
     |           |
     v           v
  +------+   +--------+    +--------+
  | Meta |   | Data 1 |    | Data 2 |  ...
  | Svc  |   | Node   |    | Node   |
  +------+   +--------+    +--------+

  Meta Service:  Cluster discovery, node registry, leader election
  Data Nodes:    Query execution, storage, compaction
  Admin Node:    Web UI, request forwarding, event streaming
  • Meta 服务 -- 管理节点查询 Meta 端点以发现所有已注册的数据节点、其角色、状态和表分配信息
  • 数据节点 -- 查询、存储操作、压缩合并和刷新命令直接转发到相应数据节点的 HTTP API
  • 事件流 -- 管理节点接收实时集群事件（节点加入、Leader 变更、表重新分配），并通过 WebSocket 流式传输到浏览器

仪表盘

仪表盘是管理控制台的首页。它提供集群健康状况和关键统计数据的高层概览。

集群健康

顶部区域显示整体集群状态：

  • 集群状态 -- 健康（所有节点响应正常）或降级（一个或多个节点不可达）
  • Meta Leader -- 当前 Meta Leader 节点 ID 及其 Raft 任期编号

节点数量卡片

四个汇总卡片显示集群中每种节点类型的数量：

卡片 显示内容
Meta 节点 Meta 服务实例总数（活跃/非活跃）
Router 节点 用于查询路由的 Router 实例总数
数据节点 数据节点总数，包含主节点/备用/退役中/已排空的明细
表 集群中已注册的表和分区总数

数据节点明细

卡片下方的详细明细显示数据节点状态：

  • 主节点 -- 接受写入和查询的活跃节点
  • 备用节点 -- 准备好在主节点故障时接管的健康节点
  • 退役中 -- 缩容期间正在迁移表的节点
  • 已排空 -- 已完成表迁移、可以安全移除的节点

快捷操作

仪表盘包含快捷操作卡片，链接到最常用的运维页面：节点管理、表概览和缩容向导。

导航

管理控制台采用侧边栏导航布局。所有页面分为两组：

主要页面

页面 路径 用途
仪表盘 / 集群健康概览、节点数量、快捷操作
查询控制台 /query Monaco SQL 编辑器，支持执行、解释和分析
存储浏览器 /storage catalog、schema、表和 Parquet 文件的树形视图
节点 /nodes 节点列表，支持角色/状态过滤和搜索
节点详情 /nodes/:id 单个节点的健康状态、统计信息和管理操作
表 /tables 表列表，包含所有权、大小和重新分配功能
缩容 /scale-in 用于集群缩容操作的多步向导
事件 /events 实时集群事件流，支持过滤

运维页面

页面 路径 用途
Meta 操作 /meta-operations Meta 集群状态、选举、分布配置
数据操作 /data-operations 跨节点批量刷新、压缩合并和缓存操作
调试工具 /debug 测试数据生成、性能分析和缓存管理
设置 /settings 管理节点配置和集群信息展示

入门指南

启动集群后，按照以下六个步骤熟悉管理控制台：

💡 前提条件

确保您的集群已运行并配置了管理节点。请参阅管理控制台配置或使用开发集群快速入门。

步骤 1：检查集群健康

在浏览器中打开 http://localhost:18080。仪表盘会自动加载，显示集群状态、Meta Leader 和节点数量。确认集群状态为健康且所有预期节点都已出现。

步骤 2：生成测试数据

在侧边栏中导航到调试工具。使用测试数据生成卡片将示例指标（CPU、内存、磁盘、网络）写入数据库。配置数据库名称、日期范围和数据点数量，然后点击生成。

步骤 3：运行第一个查询

转到查询控制台。从下拉菜单中选择您的 catalog，然后尝试一个简单的查询：

sql 
-- List all tables in the selected catalog
SHOW TABLES;

-- Query the test data you just generated
SELECT * FROM default.cpu ORDER BY time DESC LIMIT 10;

按 Ctrl+Enter（macOS 上为 Cmd+Enter）执行。结果将显示在编辑器下方的可排序、分页表格中。

步骤 4：浏览存储

打开存储浏览器并展开 catalog 树。浏览 schema 和表以查看分区详情、文件数量和存储大小。点击 Parquet 文件可检查其行组、压缩和列统计信息。

步骤 5：监控事件

访问事件页面查看实时集群事件流。节点加入、表注册和 Leader 选举等事件会实时显示。使用类型和时间过滤器缩小视图范围。

步骤 6：查看设置

检查设置页面以确认管理节点配置，包括 HTTP 端口、Meta 端点和集群信息。

核心概念

以下定义将帮助您理解管理控制台中使用的术语：

🗃

Catalog

顶层命名空间，对应写入 URI（/write/{db}）中的数据库名称。每个 catalog 包含一个或多个 schema。

📁

Schema

catalog 内的分组，由 service 标签派生（ServiceAsSchema 模式）或固定为 default（ServiceAsTable 模式）。

📅

分区

表中的数据按日期分区（date=YYYY-MM-DD）。分区支持查询时的高效时间范围裁剪。

📊

压缩合并级别

Parquet 文件通过压缩合并级别递进：L0（最近写入）、L1（首次合并）、L2+（更高级别合并）。更高级别具有更少、更大的文件，以获得更好的查询性能。

📝

WAL（预写日志）

传入的写入在存储到内存缓存之前先记录到 WAL 中以确保持久性。重启时会重放 WAL 以恢复未刷新的数据。

🌐

节点状态

数据节点在不同状态之间转换：活跃（服务流量）、非活跃（不可达）、排空中/退役中（迁移表）、已排空（可安全移除）。

🔗

表所有权

每个表被分配到一个主数据节点，负责其写入和压缩合并。可以将表重新分配到其他节点以实现负载均衡或缩容。

📈

缩容

通过退役数据节点来优雅地缩减集群规模的过程。在退役节点排空之前，表会迁移到剩余节点。

配置

管理节点通过 TOML 配置文件进行配置。以下是典型配置：

toml 
[admin]
# Enable the admin console
enabled = true

# HTTP host and port for the admin web interface
host = "0.0.0.0"
port = 18080

# Meta service endpoints for cluster discovery
# List multiple endpoints for high availability
meta_endpoints = [
    "http://127.0.0.1:6000",
    "http://127.0.0.1:6100",
    "http://127.0.0.1:6200"
]

配置选项

选项 类型 默认值 说明
enabled bool false 启用或禁用管理控制台
host string "0.0.0.0" 管理 HTTP 服务器的绑定地址
port u16 18080 管理 Web 界面的 HTTP 端口
meta_endpoints array [] 用于集群发现的 Meta 服务 URL 列表

开发集群快速启动

体验管理控制台最快的方式是使用本地开发集群：

bash 
# Start the full local cluster (meta x3, data x2, router, admin)
goreman -f dev/Procfile.cluster-local start

# Default ports:
#   Meta nodes:  6000, 6100, 6200
#   Data nodes:  5000, 5100
#   Router:      7000
#   Admin:       18080

# Open the admin console
open http://localhost:18080
