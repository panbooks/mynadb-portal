什么是 MynaDB？

正如鹩哥以其卓越的倾听和完美复述声音的能力而闻名， MynaDB 以完美的保真度捕获来自基础设施的每一个信号。 它是一款专为高吞吐量数据接入和快速分析查询而设计的时序数据库， 融合了现代数据基础设施的精华：

  • InfluxDB Line Protocol 与 OpenTelemetry - 多协议接入，支持指标、日志和链路追踪
  • Apache Parquet 与 Iceberg - 列式存储，支持多引擎目录
  • Apache DataFusion - 完整 SQL 支持，包含预测 UDF 和地理空间函数
  • Apache Arrow Flight - 高性能 gRPC 流式数据传输
  • Grafana 与 Prometheus - 原生监控集成和指标导出
  • Rust - 内存安全、零成本抽象，实现极致性能

核心特性

🔥

高性能

使用 Rust 编写，采用零拷贝反序列化，实现最大吞吐量和最低延迟。

🔎

SQL 查询

完整的 SQL 支持，包括 JOIN、聚合、窗口函数和 CTE。

🗃

列式存储

Parquet 格式提供高达 90% 的压缩率，同时保持优秀的查询性能。

🔌

兼容 InfluxDB

可直接替换 InfluxDB 写入 API，实现无缝迁移。

⚡

即时可见

通过 Memtable 立即查询新写入的数据 - 无需等待刷盘！

📈

智能缓存

多级缓存：元数据、查询结果和 SSD 上的 Parquet 文件缓存。

快速示例

bash 
# 使用 InfluxDB Line Protocol 写入指标数据
curl -X POST http://localhost:7000/write/mydb -d '
cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-east usage=82.3,temp=50
memory,host=server01 used=16.5,total=32.0
'

# 立即查询 - 无需刷盘！（Memtable 提供即时可见性）
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) as avg_cpu FROM cpu GROUP BY host'

# 使用 GROUP BY time() 进行时序聚合
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT AVG(usage) FROM cpu GROUP BY time(5m), host'
