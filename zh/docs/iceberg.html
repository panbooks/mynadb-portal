新 Apache Iceberg 集成

MynaDB 提供 Apache Iceberg REST Catalog 接口，支持对时序数据进行多引擎分析。Apache Spark、Trino、DuckDB 和 Apache Flink 等外部引擎可以通过标准 Iceberg REST Catalog API 发现和查询 MynaDB 表，无需任何数据复制或 ETL。

⚙ 多引擎分析

Iceberg 集成让您可以继续通过 MynaDB 的优化写入路径摄取数据，同时通过 Iceberg 表格式将数据湖开放给更广泛的分析生态系统。

REST Catalog API

MynaDB 实现了 Iceberg REST Catalog 规范，提供一组 HTTP 端点，供外部引擎用于发现命名空间、列出表、获取表元数据和规划扫描。

方法 端点 说明
GET /iceberg/v1/namespaces 列出所有命名空间（catalog/schema）
GET /iceberg/v1/namespaces/:ns/tables 列出命名空间内的表
GET /iceberg/v1/namespaces/:ns/tables/:table 加载表元数据（schema、分区规范、快照）
POST /iceberg/v1/namespaces/:ns/tables/:table/scan 规划表扫描（支持可选过滤器）

列出命名空间

bash 
# List all namespaces
curl http://localhost:7000/iceberg/v1/namespaces

# Response
{
  "namespaces": [
    ["mydb", "default"],
    ["mydb", "service_1"],
    ["metrics", "default"]
  ]
}

列出表

bash 
# List tables in a namespace
curl http://localhost:7000/iceberg/v1/namespaces/mydb.default/tables

# Response
{
  "identifiers": [
    { "namespace": ["mydb", "default"], "name": "cpu" },
    { "namespace": ["mydb", "default"], "name": "memory" },
    { "namespace": ["mydb", "default"], "name": "disk" }
  ]
}

加载表元数据

bash 
# Get table metadata including schema, partitioning, and snapshots
curl http://localhost:7000/iceberg/v1/namespaces/mydb.default/tables/cpu

# Response includes Iceberg table metadata:
# - table-uuid, format-version, location
# - schema (column names, types, field IDs)
# - partition-spec (date partitioning)
# - current-snapshot-id, snapshots list
# - sort-order, properties

规划表扫描

bash 
# Plan a scan with filter pushdown
curl -X POST http://localhost:7000/iceberg/v1/namespaces/mydb.default/tables/cpu/scan \
  -H "Content-Type: application/json" \
  -d '{
    "select": ["time", "host", "usage"],
    "filter": {
      "type": "and",
      "left": { "type": "gte", "column": "date", "value": "2026-01-01" },
      "right": { "type": "lt", "column": "date", "value": "2026-02-01" }
    }
  }'

# Response: list of data files to scan with statistics

配置

在 config.toml 中启用和配置 Iceberg 集成：

toml 
[iceberg]
# Enable the Iceberg REST Catalog API
enabled = true

# Catalog name exposed to external engines
catalog_name = "mynadb"

# How often to sync Parquet files to Iceberg metadata (seconds)
sync_interval_secs = 30

# Directory for Iceberg metadata files (manifests, snapshots)
metadata_location = "/data0/mynadb/iceberg-metadata"
参数 默认值 说明
enabled false 启用或禁用 Iceberg REST Catalog 端点
catalog_name "mynadb" 向外部引擎报告的 catalog 名称
sync_interval_secs 30 自动元数据同步周期的间隔时间
metadata_location "data/iceberg-metadata" Iceberg 元数据文件的存储路径
⚠ 对象存储元数据

使用 S3/MinIO/OSS 作为存储后端时，请将 metadata_location 设置为对象存储 bucket 内的路径，以便外部引擎可以从同一存储层访问数据和元数据文件。

多引擎支持

由于 MynaDB 实现了 Iceberg REST Catalog 规范，任何支持 Iceberg REST catalog 的引擎都可以直接读取 MynaDB 表。以下是常用分析引擎的配置示例。

Apache Spark

python 
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("MynaDB Iceberg") \
    .config("spark.sql.catalog.mynadb", "org.apache.iceberg.spark.SparkCatalog") \
    .config("spark.sql.catalog.mynadb.type", "rest") \
    .config("spark.sql.catalog.mynadb.uri", "http://localhost:7000/iceberg/v1") \
    .getOrCreate()

# List namespaces
spark.sql("SHOW NAMESPACES IN mynadb").show()

# List tables
spark.sql("SHOW TABLES IN mynadb.default").show()

# Query time series data
df = spark.sql("""
    SELECT host, AVG(usage) as avg_cpu, MAX(usage) as max_cpu
    FROM mynadb.default.cpu
    WHERE date >= '2026-01-01'
    GROUP BY host
    ORDER BY avg_cpu DESC
""")
df.show()

Trino

properties 
# etc/catalog/mynadb.properties
connector.name=iceberg
iceberg.catalog.type=rest
iceberg.rest-catalog.uri=http://localhost:7000/iceberg/v1
sql 
-- Query MynaDB from Trino
SELECT host, region,
       AVG(usage) AS avg_cpu,
       COUNT(*) AS sample_count
FROM mynadb.default.cpu
WHERE date >= DATE '2026-01-01'
GROUP BY host, region
ORDER BY avg_cpu DESC;

DuckDB

sql 
-- Install and load the Iceberg extension
INSTALL iceberg;
LOAD iceberg;

-- Attach MynaDB as an Iceberg REST catalog
ATTACH 'mynadb' AS mynadb (
    TYPE ICEBERG,
    ENDPOINT 'http://localhost:7000/iceberg/v1'
);

-- Query tables
SELECT host, AVG(usage) AS avg_cpu
FROM mynadb.default.cpu
WHERE date >= '2026-01-01'
GROUP BY host
ORDER BY avg_cpu DESC;

Apache Flink

sql 
-- Register MynaDB Iceberg catalog in Flink SQL
CREATE CATALOG mynadb WITH (
    'type' = 'iceberg',
    'catalog-type' = 'rest',
    'uri' = 'http://localhost:7000/iceberg/v1'
);

-- Switch to the MynaDB catalog
USE CATALOG mynadb;
USE `default`;

-- Streaming read from MynaDB
SELECT host, usage, `time`
FROM cpu
WHERE date >= '2026-01-01';

元数据同步

MynaDB 自动将其内部 Parquet 文件与 Iceberg 元数据同步。此过程在后台运行，确保外部引擎始终看到数据的一致、最新视图。

text 
┌─────────────────────────────────────────────────────────────────────┐
│                          MynaDB                                    │
│                                                                    │
│  ┌──────────────┐    ┌──────────────┐    ┌───────────────────────┐ │
│  │  Write Path  │    │  Compactor   │    │  Iceberg Sync Job     │ │
│  │              │    │              │    │                       │ │
│  │  Parquet     │───>│  Merged      │───>│  Manifest Files       │ │
│  │  Files (L0)  │    │  Files (L1+) │    │  Snapshot Metadata    │ │
│  └──────────────┘    └──────────────┘    │  Table Metadata       │ │
│                                          └───────────┬───────────┘ │
└──────────────────────────────────────────────────────┼─────────────┘
                                                       │
                            ┌──────────────────────────┼──────────┐
                            │    Iceberg Metadata      │          │
                            │                          ▼          │
                            │    metadata/                        │
                            │    ├── v1.metadata.json             │
                            │    ├── v2.metadata.json             │
                            │    ├── snap-*.avro                  │
                            │    └── manifest-*.avro              │
                            └─────────────────────────────────────┘
                                                       │
                     ┌─────────────────────────────────┼──────────┐
                     │          External Engines        │          │
                     │    ┌─────┐  ┌───────┐  ┌──────┐ │          │
                     │    │Spark│  │ Trino │  │DuckDB│ │  ......  │
                     │    └─────┘  └───────┘  └──────┘ │          │
                     └─────────────────────────────────┴──────────┘

工作原理

  1. Parquet 发现 -- 同步任务在每个 sync_interval_secs 间隔扫描 MynaDB 的数据目录，查找新的或修改过的 Parquet 文件。
  2. Schema 推断 -- 从 Parquet 文件尾部提取列名、数据类型和字段 ID，并映射为 Iceberg schema 格式。
  3. Manifest 生成 -- 新的 Parquet 文件被分组到 Iceberg manifest 文件中，记录文件路径、分区值、行数和列级统计信息。
  4. 快照提交 -- 原子性地提交新的 Iceberg 快照，指向更新后的 manifest 文件集。之前的快照会被保留以支持时间旅行查询。
  5. 元数据更新 -- 表元数据 JSON 文件随新的当前快照 ID 更新，确保外部引擎在下次刷新 catalog 时看到最新数据。
🕒 近实时

默认 sync_interval_secs = 30 时，写入 MynaDB 的新数据会在 30 秒内对外部 Iceberg 引擎可见。降低此值可获得更低延迟，但会稍微增加元数据开销。

手动同步

您可以通过 API 触发立即元数据同步：

bash 
# Force an immediate Iceberg metadata sync
curl -X POST http://localhost:7000/iceberg/v1/sync

# Response
{
  "status": "ok",
  "tables_synced": 12,
  "snapshots_created": 5,
  "duration_ms": 230
}

通过 Iceberg 查询

启用 Iceberg 集成后，外部引擎可以使用标准 Iceberg 表扫描来查询 MynaDB 数据。MynaDB 基于日期的分区直接映射到 Iceberg 分区规范，使外部引擎能够进行分区裁剪。

分区裁剪

Iceberg 元数据包含分区统计信息，允许引擎跳过不相关的日期分区：

sql 
-- Only scans Parquet files in the date=2026-02-* partitions
SELECT host, AVG(usage) AS avg_cpu
FROM mynadb.default.cpu
WHERE date >= '2026-02-01' AND date < '2026-03-01'
GROUP BY host;

时间旅行

Iceberg 快照允许您查询数据在过去某个时间点的状态：

sql 
-- Spark: query a historical snapshot
SELECT * FROM mynadb.default.cpu VERSION AS OF 123456789;

-- Spark: query as of a timestamp
SELECT * FROM mynadb.default.cpu TIMESTAMP AS OF '2026-02-01 00:00:00';

Schema 演进

当 MynaDB 在传入数据中检测到新列时，Iceberg 元数据会在下一个同步周期更新为演进后的 schema。外部引擎会自动获取新列。

sql 
-- After adding a new "temperature" field via line protocol,
-- the column appears automatically in Iceberg queries
SELECT host, usage, temperature
FROM mynadb.default.cpu
WHERE date = '2026-02-13';

跨引擎分析示例

在 Spark 中将 MynaDB 时序数据与外部数据集组合：

python 
# Read time series from MynaDB via Iceberg
cpu_df = spark.sql("""
    SELECT host, date,
           AVG(usage) as avg_cpu,
           MAX(usage) as peak_cpu
    FROM mynadb.default.cpu
    WHERE date >= '2026-02-01'
    GROUP BY host, date
""")

# Read host metadata from a data lake
hosts_df = spark.read.parquet("s3://data-lake/hosts/")

# Join time series with metadata for enriched analytics
enriched = cpu_df.join(hosts_df, "host") \
    .select("host", "region", "team", "avg_cpu", "peak_cpu")

enriched.show()
