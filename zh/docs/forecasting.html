预测分析与地理空间函数

内置预测和地理空间 UDF，用于高级时间序列分析。无需外部工具或库， 即可直接在 SQL 查询中预测未来值，并对带有位置信息的时间序列数据执行空间计算。

💡 SQL 原生分析

所有预测和地理空间函数均作为原生 UDF 实现在 DataFusion 查询引擎中。它们在进程内运行，零序列化开销，支持在标准 SQL 聚合的同时 进行实时预测和空间查询。

预测函数

MynaDB 提供了一套预测函数，可对时间序列数据进行操作以预测未来值。 这些函数分析历史模式，并按指定的周期数向前外推。

函数 说明
forecast_linear(column, periods) 简单线性回归预测。对历史数据拟合直线，并按给定周期数向前投射。
forecast_ema(column, periods, alpha) 指数移动平均预测。使用平滑因子（alpha 取值 0 到 1 之间）进行加权平均，适用于趋势敏感的预测。
forecast_arima(column, periods, p, d, q) ARIMA 模型预测。自回归积分滑动平均模型，支持可配置的阶数参数 p（AR）、d（差分）和 q（MA）。
forecast_auto(column, periods) 自动模型选择预测。评估多个模型并根据 AIC/BIC 准则选择最佳拟合。
forecast_series(column, periods, model) 使用指定模型进行预测。model 参数接受 'linear'、'ema'、'arima'、'holt_winters' 或 'seasonal'。

forecast_linear

对历史数据拟合简单线性回归（最小二乘法），并将趋势向前投射。 最适合具有稳定上升或下降趋势且季节性较低的数据。

sql 
-- 使用线性回归预测接下来 12 个小时的 CPU 使用率
SELECT
  forecast_linear(usage, 12) AS predicted
FROM cpu
WHERE host = 'server01'
GROUP BY time(1h);

forecast_ema

使用可配置平滑因子的指数移动平均。较高的 alpha （接近 1）赋予近期观测值更大权重，使预测对近期变化更敏感。 较低的 alpha（接近 0）产生更平滑、更稳定的预测。

sql 
-- 使用 EMA 预测接下来 24 个数据点（alpha=0.3，中等平滑度）
SELECT
  time_bucket('15 minutes', time) AS bucket,
  AVG(temperature) AS actual,
  forecast_ema(temperature, 24, 0.3) AS predicted
FROM sensors.default.environment
WHERE location = 'datacenter-east'
  AND time > now() - INTERVAL '7 days'
GROUP BY bucket
ORDER BY bucket;

forecast_arima

实现 ARIMA（自回归积分滑动平均）模型，非常适合具有复杂趋势和自相关性的时间序列。 参数 p、d 和 q 分别控制自回归阶数、差分阶数和滑动平均阶数。

sql 
-- ARIMA(2,1,1) 预测接下来 48 个周期的内存使用率
SELECT
  time_bucket('30 minutes', time) AS bucket,
  AVG(mem_used_percent) AS actual,
  forecast_arima(mem_used_percent, 48, 2, 1, 1) AS predicted
FROM mydb.default.system_metrics
WHERE host = 'db-primary'
  AND time > now() - INTERVAL '14 days'
GROUP BY bucket
ORDER BY bucket;

forecast_auto

自动评估多种预测模型（线性、EMA、ARIMA 变体）与输入数据的拟合程度， 并根据信息准则（AIC/BIC）选择最佳模型。当您不确定哪种模型最适合数据时， 推荐使用此函数。

sql 
-- 让 MynaDB 自动选择最佳预测模型
SELECT
  time_bucket('1 hour', time) AS bucket,
  AVG(request_rate) AS actual,
  forecast_auto(request_rate, 72) AS predicted
FROM mydb.default.api_metrics
WHERE service = 'payment-gateway'
  AND time > now() - INTERVAL '30 days'
GROUP BY bucket
ORDER BY bucket;

forecast_series

灵活的预测函数，接受模型名称作为字符串参数。当模型选择由应用逻辑驱动 或需要在相同数据上比较不同模型时，此函数非常有用。

sql 
-- 使用 Holt-Winters 预测季节性数据（如每日流量模式）
SELECT
  time_bucket('1 hour', time) AS bucket,
  AVG(active_connections) AS actual,
  forecast_series(active_connections, 168, 'holt_winters') AS predicted
FROM mydb.default.network
WHERE interface = 'eth0'
  AND time > now() - INTERVAL '60 days'
GROUP BY bucket
ORDER BY bucket;

-- 并排比较不同模型
SELECT
  time_bucket('1 hour', time) AS bucket,
  AVG(cpu_usage) AS actual,
  forecast_series(cpu_usage, 24, 'linear') AS linear_pred,
  forecast_series(cpu_usage, 24, 'ema') AS ema_pred,
  forecast_series(cpu_usage, 24, 'seasonal') AS seasonal_pred
FROM mydb.default.host_metrics
WHERE host = 'web-01'
  AND time > now() - INTERVAL '14 days'
GROUP BY bucket
ORDER BY bucket;
💡 如何选择合适的模型

Linear 最适合有明确趋势且无季节性的数据。 EMA 适合噪声较大且近期值更重要的数据。 ARIMA 处理复杂的自相关模式。 Holt-Winters 和 Seasonal 擅长处理具有重复周期性模式 的数据（如日周期或周周期）。如果不确定，使用 forecast_auto 让 MynaDB 自动选择。

地理空间函数

MynaDB 包含地理空间 UDF，用于计算距离、编码位置信息，以及对带有经纬度坐标的 时间序列数据进行空间过滤。这些函数使用 WGS84 椭球模型进行精确的地球表面计算。

函数 说明
geo_distance(lat1, lon1, lat2, lon2) 使用 Haversine 公式计算两个地理点之间的大圆距离（单位：米）。
geo_hash(lat, lon, precision) 将经纬度对编码为指定精度（1-12 个字符）的 Geohash 字符串。
geo_within(lat, lon, polygon) 如果点在给定多边形内则返回 true。多边形以 WKT 字符串格式指定（如 'POLYGON((...))' ）。
geo_bounding_box(lat, lon, min_lat, min_lon, max_lat, max_lon) 如果点在指定的边界框（矩形）内则返回 true。
geo_bearing(lat1, lon1, lat2, lon2) 计算从点 1 到点 2 的初始方位角（前方位角），单位为度，从正北顺时针测量（0-360）。

geo_distance

使用 Haversine 公式计算地球表面两点之间的大圆距离。结果以米为单位。 适用于邻近查询、地理围栏和物联网车队追踪中的行驶距离计算。

sql 
-- 查找仓库 5 公里范围内的所有车辆位置
SELECT
  time,
  vehicle_id,
  lat,
  lon,
  geo_distance(lat, lon, 37.7749, -122.4194) AS distance_m
FROM fleet.default.vehicle_positions
WHERE time > now() - INTERVAL '1 hour'
  AND geo_distance(lat, lon, 37.7749, -122.4194) < 5000
ORDER BY distance_m ASC;

geo_hash

将地理坐标编码为 Geohash 字符串，适用于空间索引、聚合附近的点以及基于区域的高效分组。 较高的精度值产生更长的字符串，代表更小的地理区域。

sql 
-- 按地理区域分组传感器读数（精度 5 约 4.9km x 4.9km 网格）
SELECT
  geo_hash(lat, lon, 5) AS region,
  COUNT(*) AS reading_count,
  AVG(temperature) AS avg_temp,
  MIN(temperature) AS min_temp,
  MAX(temperature) AS max_temp
FROM iot.default.weather_stations
WHERE time > now() - INTERVAL '24 hours'
GROUP BY region
ORDER BY avg_temp DESC;

geo_within

检测某点是否位于以 Well-Known Text (WKT) 格式定义的多边形内。 这对地理围栏应用至关重要，例如判断车辆是否已进入或离开某个定义的区域。

sql 
-- 查找当前在市中心服务区内的所有配送卡车
SELECT
  time,
  vehicle_id,
  lat,
  lon,
  speed_kmh
FROM fleet.default.vehicle_positions
WHERE time > now() - INTERVAL '5 minutes'
  AND geo_within(lat, lon,
    'POLYGON((-122.42 37.77, -122.42 37.79, -122.40 37.79, -122.40 37.77, -122.42 37.77))')
ORDER BY time DESC;

geo_bounding_box

快速的矩形包含检测，判断某点是否在给定的边界框内。 计算成本比多边形检测更低，适合在进行更精确的几何测试之前进行初步的空间过滤。

sql 
-- 将物联网传感器数据过滤到旧金山湾区边界框
SELECT
  time,
  sensor_id,
  lat,
  lon,
  pm25,
  pm10
FROM iot.default.air_quality
WHERE time > now() - INTERVAL '6 hours'
  AND geo_bounding_box(lat, lon, 37.20, -122.60, 37.90, -122.00)
ORDER BY pm25 DESC
LIMIT 100;

geo_bearing

计算从一个地理点到另一个地理点的初始方位角（前方位角）。结果以度为单位， 从正北顺时针测量（0 = 北，90 = 东，180 = 南，270 = 西）。 适用于追踪应用中的移动方向判断。

sql 
-- 计算连续车辆位置之间的行进方向
SELECT
  time,
  vehicle_id,
  lat,
  lon,
  geo_bearing(
    LAG(lat) OVER (PARTITION BY vehicle_id ORDER BY time),
    LAG(lon) OVER (PARTITION BY vehicle_id ORDER BY time),
    lat,
    lon
  ) AS heading_degrees,
  geo_distance(
    LAG(lat) OVER (PARTITION BY vehicle_id ORDER BY time),
    LAG(lon) OVER (PARTITION BY vehicle_id ORDER BY time),
    lat,
    lon
  ) AS distance_m
FROM fleet.default.vehicle_positions
WHERE vehicle_id = 'truck-42'
  AND time > now() - INTERVAL '2 hours'
ORDER BY time;

组合示例

MynaDB 分析 UDF 的真正威力在于将预测和地理空间函数组合在单个查询中使用。 以下示例展示了同时利用两种能力的高级使用场景。

按区域预测空气质量

结合基于 Geohash 的分组和时间序列预测，预测不同地理区域的 PM2.5 水平。

sql 
-- 预测每个地理区域未来 24 小时的 PM2.5 水平
SELECT
  geo_hash(lat, lon, 4) AS region,
  time_bucket('1 hour', time) AS bucket,
  AVG(pm25) AS avg_pm25,
  forecast_auto(pm25, 24) AS predicted_pm25
FROM iot.default.air_quality
WHERE time > now() - INTERVAL '30 days'
  AND geo_bounding_box(lat, lon, 37.20, -122.60, 37.90, -122.00)
GROUP BY region, bucket
ORDER BY region, bucket;

车队到达时间预测

结合距离计算和速度趋势预测，估算配送车辆的到达时间。

sql 
-- 预测活跃配送的剩余行程时间
SELECT
  vehicle_id,
  lat,
  lon,
  geo_distance(lat, lon, dest_lat, dest_lon) AS remaining_distance_m,
  speed_kmh,
  forecast_ema(speed_kmh, 6, 0.5) AS predicted_speed,
  geo_distance(lat, lon, dest_lat, dest_lon) / (forecast_ema(speed_kmh, 6, 0.5) * 1000.0 / 3600.0) AS eta_seconds,
  geo_bearing(lat, lon, dest_lat, dest_lon) AS heading_to_dest
FROM fleet.default.vehicle_positions v
JOIN fleet.default.active_deliveries d
  ON v.vehicle_id = d.vehicle_id
WHERE v.time > now() - INTERVAL '30 minutes'
  AND d.status = 'in_transit'
ORDER BY eta_seconds ASC;

地理围栏容量预测

结合空间过滤和容量预测，预测特定数据中心区域内服务器的资源利用率。

sql 
-- 预测美西区域服务器的磁盘使用率
SELECT
  host,
  geo_hash(datacenter_lat, datacenter_lon, 3) AS dc_region,
  time_bucket('1 day', time) AS bucket,
  AVG(disk_used_percent) AS current_usage,
  forecast_linear(disk_used_percent, 30) AS predicted_30d,
  CASE
    WHEN forecast_linear(disk_used_percent, 30) > 90 THEN 'CRITICAL'
    WHEN forecast_linear(disk_used_percent, 30) > 75 THEN 'WARNING'
    ELSE 'OK'
  END AS capacity_status
FROM mydb.default.host_metrics
WHERE time > now() - INTERVAL '90 days'
  AND geo_within(datacenter_lat, datacenter_lon,
    'POLYGON((-125.0 32.0, -125.0 49.0, -104.0 49.0, -104.0 32.0, -125.0 32.0))')
GROUP BY host, dc_region, bucket
ORDER BY predicted_30d DESC;

带空间上下文的异常检测

通过将实际读数与预测值进行比较，并结合空间邻近数据，识别温度传感器异常。

sql 
-- 检测温度读数与预测值的异常偏差，按区域分组
SELECT
  sensor_id,
  geo_hash(lat, lon, 6) AS micro_region,
  time_bucket('15 minutes', time) AS bucket,
  AVG(temperature) AS actual_temp,
  forecast_ema(temperature, 1, 0.2) AS expected_temp,
  ABS(AVG(temperature) - forecast_ema(temperature, 1, 0.2)) AS deviation,
  geo_distance(lat, lon, 37.7749, -122.4194) AS distance_from_center
FROM iot.default.temperature_sensors
WHERE time > now() - INTERVAL '7 days'
  AND geo_bounding_box(lat, lon, 37.70, -122.50, 37.85, -122.35)
GROUP BY sensor_id, micro_region, bucket, lat, lon
HAVING ABS(AVG(temperature) - forecast_ema(temperature, 1, 0.2)) > 5.0
ORDER BY deviation DESC
LIMIT 50;
💡 性能提示

在组合地理空间过滤与预测函数时，请将空间过滤条件 （geo_bounding_box、geo_within）放在 WHERE 子句中， 以在预测计算运行之前缩小数据集。预测函数受益于通过时间范围过滤的分区裁剪， 因此请始终包含 time > now() - INTERVAL '...' 谓词。
