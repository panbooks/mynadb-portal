管理控制台

基于 Web 的管理界面，用于管理和监控您的 MynaDB 集群。管理控制台提供统一的仪表盘，支持数据查询、存储浏览、集群节点管理和查询历史跟踪——一切尽在浏览器中完成。

概述

MynaDB 管理控制台是一个独立的 Web 应用程序，通过 Meta 服务连接到您的集群。它提供对数据库、表、分区和集群健康状态的全面可视化，以及用于交互式数据探索的强大 SQL 查询编辑器。

📝

SQL 查询控制台

内置 Monaco 编辑器，支持 SQL 语法高亮、自动补全和表格化结果展示。直接在浏览器中执行、解释和分析查询。

🗃

存储浏览器

通过树形视图浏览您的数据层级结构，包括 catalog、schema、表和分区。可检查 Parquet 文件详情和管理分区。

🌐

集群管理

实时监控节点状态和健康情况。查看集群事件日志并进行过滤，追踪节点加入、离开和 Leader 选举事件。

🕐

查询历史

保存常用查询以便快速访问。浏览包含执行时间详情的历史记录，查看过去查询的执行计划。

配置

在 config.toml 中启用和配置管理控制台。管理节点作为独立进程运行，连接到一个或多个 Meta 服务端点进行集群发现，并将请求转发到相应的数据节点。

toml 
[admin]
# Enable the admin console
enabled = true

# Port for the admin web interface
port = 18080

# Meta service endpoints for cluster discovery
meta_endpoints = [
    "http://127.0.0.1:6000",
    "http://127.0.0.1:6100",
    "http://127.0.0.1:6200"
]

配置选项

选项 类型 默认值 说明
enabled bool false 启用或禁用管理控制台
port u16 18080 管理 Web 界面的 HTTP 端口
meta_endpoints array [] 用于集群发现的 Meta 服务 URL 列表
💡 Meta 服务发现

管理控制台使用 Meta 服务来发现集群中的所有数据节点。至少需要提供一个 Meta 端点。为了实现高可用，建议列出多个 Meta 端点，以便在某个端点不可用时进行故障转移。

SQL 查询控制台

SQL 查询控制台是与数据交互的主要界面。它提供了一个功能完整的代码编辑器，由 Monaco（与 VS Code 使用相同的编辑器）驱动，配备了执行、解释和分析查询的专用控件。

功能特性

  • Monaco 编辑器 -- 完整的 SQL 语法高亮、括号匹配和自动缩进
  • 执行查询 -- 运行 SQL 查询并在可排序、分页的表格中查看结果
  • 执行计划 -- 查看查询的逻辑和物理执行计划
  • 分析查询 -- 查看分区裁剪详情、文件数量和优化摘要
  • 导出结果 -- 将查询结果下载为 CSV 或 JSON 格式
  • 保存查询 -- 保存和整理常用查询以便快速访问
  • 查询历史 -- 自动记录每次执行的查询，包含执行时间和行数

数据库和 Schema 选择

使用控制台顶部的 catalog 和 schema 下拉菜单设置查询上下文。控制台会自动从集群中发现可用的数据库和 schema：

sql 
-- With catalog "mydb" selected, you can use short names:
SELECT * FROM default.cpu WHERE time > now() - INTERVAL '1 hour';

-- Or fully qualified names work from any context:
SELECT * FROM mydb.default.cpu WHERE time > now() - INTERVAL '1 hour';

键盘快捷键

快捷键 操作
Ctrl+Enter / Cmd+Enter 执行当前查询
Ctrl+E / Cmd+E 查看当前查询的执行计划
Ctrl+S / Cmd+S 保存当前查询
Ctrl+Shift+A / Cmd+Shift+A 分析当前查询

存储浏览器

存储浏览器提供了 MynaDB 集群中所有数据的层级视图。浏览 catalog、schema、表和分区，检查存储详情并管理您的数据。

功能特性

  • 树形导航 -- 在可折叠的树形视图中展开 catalog、schema 和表
  • 表详情 -- 查看表 schema、列类型、行数和存储大小
  • 分区列表 -- 浏览日期分区，包含每个分区的文件数量和大小
  • Parquet 文件详情 -- 检查单个 Parquet 文件的行组、压缩和列统计信息
  • 分区管理 -- 直接从界面删除单个分区或日期范围
  • 触发压缩合并 -- 为特定表或整个 catalog 启动压缩合并

存储层级结构

text 
Cluster
 +-- mydb                          (catalog)
 |    +-- default                  (schema)
 |    |    +-- cpu                 (table)
 |    |    |    +-- date=2024-01-07  (partition, 3 files, 12.4 MB)
 |    |    |    +-- date=2024-01-08  (partition, 2 files, 8.1 MB)
 |    |    |    +-- date=2024-01-09  (partition, 1 file, 3.2 MB)
 |    |    +-- memory              (table)
 |    +-- service_1                (schema)
 |         +-- disk                (table)
 +-- otel                          (catalog)
      +-- default                  (schema)
           +-- otel_metrics        (table)
           +-- otel_logs           (table)

压缩合并控制

在存储浏览器中，您可以在不同级别触发压缩合并：

bash 
# The admin console triggers these API calls on the data node:

# Compact an entire catalog
POST /compact/mydb

# Compact a specific measurement
POST /compact/mydb/cpu

集群管理

集群管理视图让您实时了解 MynaDB 集群中每个节点的状态。监控节点健康、追踪事件并快速识别问题。

节点列表

节点列表显示所有已注册的数据节点及其当前状态：

列 说明
节点 ID 节点的唯一标识符
地址 HTTP 和 Flight 端点地址
状态 在线、离线或降级
角色 Leader 或 Follower（用于后台任务分配）
运行时间 节点最后一次向 Meta 服务注册以来的时间
表数量 分配给此节点的表数量

事件日志

事件日志实时捕获集群级别的事件。使用过滤控件按类型或时间范围筛选事件：

  • 节点加入 -- 数据节点向 Meta 服务注册
  • 节点离开 -- 数据节点注销或超时
  • Leader 选举 -- 为后台任务选举了新的 Leader
  • 表创建 -- 集群中注册了新表
  • 压缩合并开始/完成 -- 压缩合并任务开始或结束
  • 数据保留执行 -- 过期分区被删除

健康检查

管理控制台定期轮询每个数据节点的 /health 端点，并显示聚合的集群健康状态。如果节点在配置的超时时间内未能响应，将被标记为降级；如果连续多次健康检查失败，则标记为离线。

⚠ 节点连通性

管理控制台必须能够直接访问每个数据节点的 HTTP 端口。请确保管理节点和数据节点之间没有防火墙规则或网络策略阻止流量。

访问控制台

管理节点运行后，在浏览器中打开控制台：

text 
http://localhost:18080

默认端口为 18080。如果您在配置中更改了端口，请使用配置的值。

认证

如果数据节点启用了认证，管理控制台将在首次访问时提示您输入凭据。数据节点配置中 [auth] 部分的用户名和密码同样适用：

toml 
# Data node auth configuration
[auth]
enabled = true
username = "admin"
password = "secret"
🔒 安全建议

在生产环境中，始终启用认证，并通过 HTTPS 访问管理控制台——将其部署在 Nginx 或 Caddy 等反向代理之后进行 TLS 终止。更多详情请参阅认证与 RBAC 页面。

开发集群快速启动

在本地开发集群中运行管理控制台：

bash 
# Start the full local cluster (meta, data, router, admin)
goreman -f dev/Procfile.cluster-local start

# Default ports:
#   Meta nodes:  6000, 6100, 6200
#   Data nodes:  5000, 5100
#   Router:      7000
#   Admin:       18080

# Open the admin console
open http://localhost:18080

事件监控

事件页面提供通过 WebSocket 传输的集群事件实时流。监控节点加入、离开、Leader 选举、表重新分配等事件。按类型、时间范围和来源过滤事件，并导出为 JSON 或 CSV 进行分析。

有关所有 13 种事件类型的完整列表和详细使用说明，请参阅集群运维指南中的事件监控部分。

缩容操作

缩容页面提供了一个引导式三步向导，用于优雅地减少集群中的数据节点数量。可选择最少负载、先进先出、后进先出或高可用单元等策略，预览迁移计划，并通过实时进度跟踪执行操作。

有关缩容向导的完整操作指南，请参阅集群运维指南中的缩容操作部分。

调试工具

调试工具页面提供了测试数据生成、CPU 和堆内存分析以及缓存管理等工具。生成示例时序指标、下载性能分析快照，或在数据节点上预热和清除缓存。

有关每个调试工具的详细说明，请参阅集群运维指南中的调试工具部分。
