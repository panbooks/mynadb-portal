Grafana 集成

使用 Grafana 仪表板可视化您的时序数据。MynaDB 通过多种数据源选项与 Grafana 无缝集成，包括 InfluxDB 兼容端点、通过 MySQL 协议的原生 SQL，以及用于运维可视化的内置监控仪表板。

数据源配置

按照以下步骤使用 InfluxDB 数据源插件将 Grafana 连接到 MynaDB：

  1. 打开 Grafana 并导航到数据源
    在 Grafana 实例中前往 配置 → 数据源 → 添加数据源。
  2. 选择 InfluxDB 数据源类型
    在数据源列表中搜索 InfluxDB 并选择它。MynaDB 提供 InfluxDB 兼容的查询端点。
  3. 配置连接 URL
    将 URL 设置为 MynaDB 实例的 HTTP 地址：
    text 
    http://your-mynadb-host:7000
  4. 设置数据库名称
    在 Database 字段中输入您的数据库（catalog）名称。这对应于 MynaDB 写入和查询端点中使用的 :db 参数（例如 mydb）。
  5. 配置认证（如已启用）
    如果 MynaDB 启用了认证，请开启 Basic Auth 并提供在 config.toml 的 [auth] 部分配置的用户名和密码。
  6. 保存并测试
    点击 保存并测试。Grafana 将验证与 MynaDB 的连接。

InfluxDB 兼容端点

MynaDB 提供多个查询端点，Grafana 可根据您偏好的查询语言选择使用：

端点 协议 说明
/api/v2/query Flux InfluxDB v2 兼容的 Flux 查询端点。当 Grafana 配置为 Flux 模式的 InfluxDB 数据源时使用此端点。
/api/v1/query InfluxQL InfluxDB v1 兼容的 InfluxQL 查询端点。适用于传统仪表板或更简单的查询语法。
/query/:db 原生 SQL 由 Apache DataFusion 驱动的 MynaDB 原生 SQL 端点。提供完整的 SQL 功能，包括 JOIN、窗口函数和 CTE。

内置仪表板

MynaDB 自带一个可从管理控制台访问的 Grafana 风格监控仪表板。此内置仪表板无需单独安装 Grafana 即可提供实时运维可视化。

仪表板包含以下面板：

  • 数据接入速率 — 跟踪每秒写入的数据点数，按数据库和 measurement 分类。有助于识别写入吞吐量瓶颈和流量峰值。
  • 查询延迟 — 显示 p50、p95 和 p99 查询执行时间。监控查询性能随时间的变化，突出显示可能需要优化的慢查询。
  • 存储大小 — 显示所有 catalog 和 schema 的总磁盘使用量，按压缩级别（L0、L1、L2）分类。对容量规划和跟踪数据增长很有用。
  • 压缩合并进度 — 可视化活跃的压缩合并任务、已合并的文件、回收的字节数和压缩合并积压量。确保后台维护跟上写入量。

访问内置仪表板：

text 
http://your-mynadb-host:7000/admin/dashboard

Grafana 中的示例查询

在 Grafana 中使用 MynaDB 的原生 SQL 端点时，您可以使用 Grafana 内置的模板变量来创建动态的、具有时间感知能力的仪表板。以下是常见的查询模式。

按时间过滤的聚合

sql 
SELECT
  time_bucket('$__interval', time) AS time,
  avg(cpu_usage) AS avg_cpu,
  max(cpu_usage) AS max_cpu
FROM mydb.default.system_metrics
WHERE $__timeFilter(time)
GROUP BY time_bucket('$__interval', time)
ORDER BY time

多主机对比

sql 
SELECT
  time_bucket('$__interval', time) AS time,
  host,
  avg(memory_used_percent) AS avg_memory
FROM mydb.default.system_metrics
WHERE $__timeFilter(time)
  AND host IN ($host)
GROUP BY time_bucket('$__interval', time), host
ORDER BY time

数据接入速率监控

sql 
SELECT
  time_bucket('$__interval', time) AS time,
  count(*) / EXTRACT(EPOCH FROM INTERVAL '$__interval') AS points_per_second
FROM mydb.default.measurements
WHERE $__timeFilter(time)
GROUP BY time_bucket('$__interval', time)
ORDER BY time

带变量的 Top-N 查询

sql 
SELECT
  host,
  avg(cpu_usage) AS avg_cpu,
  max(cpu_usage) AS peak_cpu
FROM mydb.default.system_metrics
WHERE $__timeFilter(time)
GROUP BY host
ORDER BY avg_cpu DESC
LIMIT $topN

支持以下 Grafana 宏：

宏 说明 展开示例
$__timeFilter(time) 根据仪表板选定的时间范围展开为时间范围过滤器 time >= 1704067200000000000 AND time <= 1704153600000000000
$__interval 根据仪表板时间范围和面板宽度自动计算的聚合间隔 1m、5m、1h
$__from / $__to 选定仪表板时间范围的开始和结束时间戳 1704067200000

MySQL 数据源替代方案

作为 InfluxDB 数据源的替代方案，您可以使用 MySQL 数据源将 Grafana 连接到 MynaDB。MynaDB 在端口 7002 上实现了 MySQL 线协议，允许标准 MySQL 客户端和工具（包括 Grafana）直接执行 SQL 查询。

配置步骤

  1. 在 MynaDB 中启用 MySQL 协议
    确保在 config.toml 中启用了 MySQL 协议：
    toml 
    [server]
mysql_enabled = true
mysql_port = 7002
  2. 在 Grafana 中添加 MySQL 数据源
    前往 配置 → 数据源 → 添加数据源 并选择 MySQL。
  3. 配置连接
    text 
    Host:     your-mynadb-host:7002
Database: mydb
User:     admin      (if auth is enabled)
Password: secret     (if auth is enabled)
  4. 保存并测试连接
    点击 保存并测试 以验证连接。

MySQL 数据源的优势

  • 完整的 SQL 支持，包括 JOIN、子查询、CTE 和窗口函数
  • Grafana 原生 SQL 查询编辑器，带自动补全
  • 兼容 Grafana 告警规则
  • 无需安装额外插件

MySQL 数据源查询示例

sql 
SELECT
  UNIX_TIMESTAMP(time) AS time_sec,
  avg(value) AS avg_value,
  measurement
FROM cpu
WHERE time BETWEEN FROM_UNIXTIME($__from / 1000) AND FROM_UNIXTIME($__to / 1000)
GROUP BY time_bucket('$__interval', time), measurement
ORDER BY time_sec
