MynaDB
时序数据库

您的基础设施正在说话，开始倾听。

倾听。存储。监控。

基于 Rust 构建的高性能时序数据库。通过 InfluxDB Line Protocol 或 OpenTelemetry 接入数据， 以列式 Parquet 格式存储并支持 Apache Iceberg，使用 Apache DataFusion 驱动的 SQL 进行查询。 多协议接入、流式查询和 Grafana 原生仪表盘。

▶ 快速开始 📖 阅读文档 
# 使用 InfluxDB Line Protocol 写入数据
curl -X POST http://localhost:7000/write/mydb \
  -d 'cpu,host=server01,region=us-west usage=75.5,temp=45'

# 使用 SQL 查询
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) FROM cpu GROUP BY host'

快速开始

60 秒内开始倾听。

bash 
# 1. 构建并运行
cargo build --release
./target/release/mynadb

# 2. 写入一些数据
curl -X POST http://localhost:7000/write/mydb -d '
cpu,host=server01,region=us-west usage=75.5,temp=45
cpu,host=server02,region=us-east usage=82.3,temp=50
memory,host=server01 used=16.5,total=32.0
'

# 3. 立即查询！（无需刷盘 - Memtable 提供即时可见性）
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) as avg_cpu FROM cpu GROUP BY host'

# 结果：
# [{"host":"server01","avg_cpu":75.5},{"host":"server02","avg_cpu":82.3}]

# 4. 使用 GROUP BY time() 进行时间序列聚合
curl 'http://localhost:7000/query/mydb' \
  -d 'SELECT host, AVG(usage) FROM cpu GROUP BY time(5m), host'
